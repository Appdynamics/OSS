1. Introduction

Please familiarize yourself with the jmxcmd overview.

JmxCmd supports facilities for adding additional functionality. These facilities include:

  • protocol providers -- add custom protocols and/or vendor specific tweaks to existing protocols
  • commands -- add new commands (this functionality is inherited from JCmd)

To add new commands, see the JCmd optional documentation.

2. Protocol providers

2.1 Providers Overview

Not all JMX MBean servers use the standard JSR 160 (Remote JMX) Connectors, and even if they do, they can still require their own product-specific information, such as authentication information. To work around both of these issues, the providers command allows the additional of pluggable modules that know how to "speak" to the vendor's server. By default, jmxcmd supports the standard JMXMP Connector.

Implementing a provider is a simple affair; in most cases it should require only a few pages of code, at most. The idea is that the command framework provides all the parameters provided by the caller to your provider, which need only concern itself with establishing the connection, subject to the information provided by the provided parameters.

If a provider has been implemented for you, and you wish to use it, you must configure jmxcmd to use it via the providers command. Once configured, the provider is remembered so that you do not have to configure it again. The jar file for the provider will need to be in the classpath. Once configured, providers are equal with any other protocol type and connections are made using the connect command.

2.2 Protocol provider implementation notes

To write a provider, you must implement com.sun.cli.jmxcmd.spi.JMXConnectorProvider. Your provider will also need to import com.sun.cli.jmxcmd.spi.JMXConnectorProviderInfo. A good sample of how to implement a provider is com.sun.cli.jmxcmd.spi.JMXMPDefaultConnectorProvider. Clicking on a classname within the Javadoc will show the source code.
