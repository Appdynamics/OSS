1. Introduction

Please familiarize yourself with the jcmd overview.

JmxCmd supports pluggable commands. Implementation notes follow.

Optional commands, including a sample command, may be loaded by running the 'add-optional' script.

 

2. Adding Commands

2.1 Commands Overview

New commands may be added or removed to/from jcmd. If you have a jar file containing commands, you can configure jcmd to use them via the add-cmd-source command. Once configured, jcmd remembers this configuration.

Commands are loaded explicitly; none are built-in. When you use JCmd, the so-called "built-in" commands are loaded only because code runs that causes them to load. The JCmd framework uses CmdMgrImpl by default unless the --cmd-mgr-classname option specifies a different CmdMgr. For example, JmxCmd uses a different CmdMgr to load additional commands.

2.2 Commands implementation notes

2.2.1 Sample command

For a working command, see the sample command com.sun.cli.jcmd.framework.SampleCmd and com.sun.cli.jcmd.framework.SampleCmdSource. The source code is linked into the Javadoc if you wish to see it. You can load and execute this sample command as follows:

jmxcmd> add-cmd-source com.sun.cli.jcmd.framework.SampleCmdSource
Added: com.sun.cli.jcmd.framework.SampleCmd, names = sample-cmd
jmxcmd> sample-cmd -?
demonstrate a sample command
sample-cmd [--boolean|-b] [--integer|-i <any-integer>] [--string|-s <any-string>] [--antisocial|-a]
[value[ value]*]

Demonstrates a sample command.
Also known as: sample-cmd

2.2.2 Framework classes to use

The minimal requirement for writing a new command is to implement com.sun.cli.jcmd.framework.Cmd and to implement a CmdSource which provides the command(s) classes to load. In implementing your commands, you will want to refer to the JavaDoc for the command framework.

For most commands you will want to extend com.sun.cli.jcmd.framework.CmdBase or com.sun.cli.jmxcmd.cmd.JMXCmd (which extends CmdBase). By extending one of these classes, you will inherit many useful facilities that do most of the work you, leaving it to you to implement just your application-specific functionality..
