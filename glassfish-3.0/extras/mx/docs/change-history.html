JCmd/JmxCmd Change History

 

Table is in reverse-chronological order.

1.0b4 (head rev)

Date Change
15 March 2004 added build-info document in preparation for doing ant-based build.
12 March 2004

added register-services command to load relation and timer services

modified StringifierRegistry code to better factoring
11 March 2004 First pass at support for JSR 77 types, and mapping them into OpenMBeans (mainly as research for MBean API).
04 March 2004 Added SASL DIGEST-MD5 support to the connect, load-jmxmp-connector commands, and tested.
01 Mar 2004 Created support classes for I18N support, modified CmdBase to use them, tested them. More I18N work to follow as time allows.
26 Feb 2004 Modified generate-mbeans command to generate interfaces that have "final" in front of each parameter.
26 Feb 2004 Updated s1as8 appserve code to support the change in ObjectNames
24 Feb 2004 Added initial support for BSF (Bean Scripting Framework). js.jar and bsf.jar must be present for it to function and they are not built into the binaries yet. This feature may or may not stay, and so far should be considered exploratory only.
23 Feb 2004 [back from vacation]. Updated documentation on how to build your own CLI, add commands, etc, based on feedback.
10 Feb 2004 Added SecurityCmd; a work in progress to help understand needs for PE/SE/EE. Supplies add-connection-cert, dump-keystore, list-keystore-aliases commands. Not officially part of jmxcmd yet.
05 Feb 2004

Modified help command to understand the overall name for a Cmd (in addition to each individual name).

 

1.0b3 released 05 Feb 2004

Date Change
05 Feb 2004

Added planned-improvements.html
05 Feb 2004

Added material on the connect command and on implementing a Provider.
30 Jan 2004 Fixed regression where output from 'get' command was not alphabetized. Now the output is alphabetized.
30 Jan 2004 Finished implementing MBeanProxyHandler and wrote a unit test for it (more to come). Now takes a ConnectionSource.
29 Jan 2004 Added MBeanProxyHandler and MBeanProxyFactory to support creating proxies to MBeans that have Attributes with names that are not legal Java identifiers.
29 Jan 2004 Modified generate-mbeans to deal with Attributes that have names that are not legal Java identifiers.
26 Jan 2004 Added 'generate-mbeans' command to generate MBean interface classes (java source) from MBeanInfo. Also modified inspect command to have a --java option to display the interface.
15 Jan 2004 Upon startup a script is executed given by the environment variable STARTUP_SCRIPT (if present)
15 Jan 2004 output of 'env' command is now sorted
15 Jan 2004 'source' command now prints a blank line between each command
15 Jan 2004 RMI Connector can now be loaded, but no support for TLS yet.
13 Jan 2004 username/password now works for security. SSL/TLS yet to be tested
12 Jan 2004 Added SASL/TLS code (not done yet)
12 Jan 2004 Fixed bug in set command that would throw an ArrayIndexOutOfBoundsException if no operands were specified

 

1.0b2

Date Change
09 Jan 2004 Listen command now warns if no objects are being listened to. Also, it displays the MBeans that are being listened to.
09 Jan 2004 added support for "verbose" environment variable. If set to "true", then it's the same as specifying --verbose
09 Jan 2004 updated version to 1.0 beta 1+ (until I call it beta 2)
09 Jan 2004

fixed history bug that wasn't unescaping characters when the history was read from a file. Also, when history is displayed tab, newline and return are show as \t, \n, \r

09 Jan 2004 source command now echoes the lines if --verbose is specified
09 Jan 2004 Fixed repeat command so that it didn't stack overflow with nested repeat commands of certain types, and also fixed it so that a repeated repeat command is relative to its original execution point. Also fixed repeat command to propogate the error code of the repeated command.
08 Jan 2004 updated documentation. In particular, added discussion of how to eliminate the SSL/TLS overhead by using jxmcmd as a proxy server
06 Jan 2004 changed source command to have --continue-on-error instead of --stop-on-error
06 Jan 2004 modified load-jmxmp-connector, load-rmi-connector, load-html-adapter to make object name optional
06 Jan 2004 modified run-tests command so that it throws a CmdException if a unit test fails
06 Jan 2004 fixed bug in stop-mbean-server command (required exactly one operand)
06 Jan 2004 fixed bug in ! command (repeat command for example "!g" )
06 Jan 2004 fixed outdated command names in start-test-mbs.jmxcmd, added test-all.jmxcmd
06 Jan 2004 modified inspect command so that summary is not displayed by default when inspecting other things. Added a --summary option
06 Jan 2004 fixed problems with help command
19 Dec 2003 Modified help command so that when invoked with no arguments, the listing produced is friendlier in its format, with each subcmd listed under its overall grouping.
19 Dec 2003 Fixed bug that resulted in help showing commands in improper order.
19 Dec 2003 Updated documents.
18 Dec 2003 Removed getName() routine from all CmdHelp implementations that didn't need it.
18 Dec 2003 Removed user/password output from s1ashttp provider.
18 Dec 2003 Added get-default-domain command.
18 Dec 2003 Added code that checks for min, max operands.

1.0b1

Date Change
18 Dec 2003 Updated site to now offer specific versions as well as head rev.
18 Dec 2003 Incorporated the latest code for S1AS8 connector.
18 Dec 2003 Revised the help classes to take advantage of the new CmdInfos class. This significantly reduces the complexity of providing help.
16 Dec 2003 Overhauled the way command information is setup and the way the help is generated so as to minimize the hard-coding and provider for better generic handling and easier command definition.
15 Dec 2003 Monitor command now accepts any wildcard expression for attributes in addition to the previous *, *r, *w expressions.
15 Dec 2003 Updated help and docs for listen, monitor commands
15 Dec 2003 Made command aliases persistent
15 Dec 2003 Updated jcmd-commands.html
15 Dec 2003 Make the list-cmd-aliases command emit the aliases in sorted order.
15 Dec 2003 Eliminated short command names (connect, c), etc and instead implemented a generic facility for command aliasing via the create-command-alias/delete-command-alias/list-command-aliases commands. These aliases are not yet persisted.
15 Dec 2003 changed create/delete/show/resolve-aliase(s) to create-target-alias, etc.
15 Dec 2003 Documented that list-aliases accepts prefixes
15 Dec 2003 Fixed show/resolve/delete-alias to require an operand
12 Dec 2003 Updated setenv, help, configure, echo, source commands to have less hard-coding of Strings for help.
12 Dec 2003 Modified list-aliases command to display only those aliases that start with the operand(s), or all aliases (as before) if there are no operands.
12 Dec 2003 Coalesced CreateAliasCmd, DeleteAliasCmd, ResolveAliasCmd, ListAliasesesCmd into AliasesCmd
12 Dec 2003 Modified source command to take a --stop-on-error option
12 Dec 2003 Modified env command so that any operands are used as prefixes. Any variables matching any prefix is display. As before, specifying no operands displays all usual variables. Also added the --all option to the syntax (was missing).
12 Dec 2003 Modified TokenizerImpl to take a TokenizerParams instead of separate parameters. Added a parameter to specify that illegal escape sequences should be emitted literally, rather than throwing an exception.
12 Dec 2003 Added --warnings-off option to validate-mbeans
12 Dec 2003 Fixed validate-mbeans so that it does not require any operands; default target will be used if none are specified. If there is no default target, then all MBeans ("*") is assumed.
12 Dec 2003 Fixed problem where if the history file or aliases file did not exist, an error message would be issued. This was confusing, especially with a new install. An error message is issue only if a file exists and there is a problem reading it.
11 Dec 2003 Changed some failures into warnings in validate-mbeans; those that involve non-serializable object.
11 Dec 2003 Remove JMXIOException because it required extra code to understand the original type of IOException ( NotSerialiazableException for example).
